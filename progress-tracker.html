🧾 Day 2 — Accounts Payable Controller + Aging Report

📌 What I Built Today

  • ✅ Built Invoice model — VendorName, Amount, InvoiceDate, DueDate, Status, IsOverdue computed property
  • ✅ Built AccountsPayableController with full CRUD and workflow endpoints
  • ✅ GET /api/AccountsPayable — list all invoices
  • ✅ GET /api/AccountsPayable/overdue — filter invoices past due date
  • ✅ GET /api/AccountsPayable/status/{status} — filter by Pending / Approved / Paid
  • ✅ POST /api/AccountsPayable — create new invoice (auto-sets status to Pending)
  • ✅ PUT /api/AccountsPayable/{id}/approve — approve a pending invoice
  • ✅ PUT /api/AccountsPayable/{id}/pay — mark approved invoice as paid
  • ✅ GET /api/AccountsPayable/aging — aging report with totals by status and overdue breakdown

🧠 Key Learnings

  • ✔ Computed properties in C# — IsOverdue uses get { return ... } to derive value from DueDate
  • ✔ Multi-step workflow enforcement — approve before pay, guard clauses with BadRequest
  • ✔ Anonymous objects — returning shaped JSON without creating a separate class
  • ✔ LINQ .Where, .Count, .Sum chained together to build an aging report in one block
  • ✔ Route parameters — {id}/approve pattern to action on a specific resource

⚠️ Challenges Faced

  • ❌ Invoice model not in Models folder — had to create it separately as untracked file
  • ✅ Added Invoice.cs to backend/FundFlowNXT.API/Models/ and wired up correctly

🚀 Next Steps

  • ➡ Commit AccountsPayableController and Invoice model to GitHub
  • ➡ Build Grants and Budgeting controllers
  • ➡ Start Angular frontend — Week 2 learning plan
  • ➡ Discuss dev database access with Satya

🎯 Outcome

Built the Accounts Payable module with a complete 3-stage approval workflow (Pending → Approved → Paid), overdue detection, status filtering, and an aging report endpoint — all in a single controller. Mirrors real-world AP processes used in FinancialEdge NXT.
